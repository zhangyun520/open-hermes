# RFCs / Request for Comments

RFCs are the narrow trunk for important Jellyfish design changes.

Use RFCs when a proposal affects protocols, governance, privacy boundaries, public commons promotion, benchmarks, or major product direction.

## Process

1. Open an issue or discussion with a short idea, failure case, benchmark, or design note.
2. If it has broad impact, draft an RFC in this folder.
3. Collect review from contributors, triagers, maintainers, and safety reviewers.
4. Update `CONTRIBUTORS.md` for accepted important contributions.
5. Keep non-canonical alternatives visible when they are useful.

## Template

```md
# RFC NNNN: Title

## Summary
## Motivation
## Proposal
## Safety and Privacy
## Benchmark / Verification
## Open Questions
```
