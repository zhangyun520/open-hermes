# Co-Creation Protocol / 共创协议

Jellyfish is an open co-creation system for human-AI symbiosis.

The protocol is **wide entrance, narrow trunk**:

- Wide entrance: anyone can submit ideas, failure cases, benchmark scenarios, code, documentation, research notes, and product designs.
- Narrow trunk: only reviewed, redacted, reproducible, and safe contributions become canonical project material.

## Contribution flow

1. Submit through an issue, discussion, RFC, or pull request.
2. Triage for type, risk, privacy, and reproducibility.
3. Redact private raw data before public review.
4. Add tests, benchmark criteria, or review notes where relevant.
5. Record important accepted contributions in `CONTRIBUTORS.md`.

## First-class contribution types

- Ideas create new possible paths.
- Failure cases reveal residuals and safety gaps.
- Benchmark scenarios make progress measurable.
- Code and documentation turn protocols into usable tools.
- Research notes and product designs widen the commons.

## Safety defaults

- Do not submit secrets, private raw data, or sensitive logs.
- Route high-risk items to human review.
- Do not place private material into public cache.
- Keep AI-generated canonical changes under human review.
