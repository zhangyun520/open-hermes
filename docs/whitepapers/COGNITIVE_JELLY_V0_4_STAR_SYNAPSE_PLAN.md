# 认知水母白皮书 v0.4｜星链突触计划

## Cognitive Jelly: 从残差操作系统到分布式星云突触网络

## 1. 摘要

认知水母 v0.4 将项目从“残差回收与技能缓存系统”推进到“星链突触计划”：把经过脱敏、授权、结构化、审核、验证和可复现评测的高质量残差肥料，转化为可验证的小突触、LoRA、Adapter、路由器、评测器、技能块、领域小模型和安全块。

本白皮书不提出真实训练网络，也不允许原始隐私数据进入训练。它定义协议、类型、mock 流程、治理边界和 MVP 路线，让未来的训练生态先有准入、评测、剪枝和伦理，再谈规模。

## 2. 核心公式：C + D = 1

`C` 是 closure / 闭合度，代表系统已经理解、验证、复用或压缩的部分。

`D` 是 residual / 残差，代表仍未被理解、验证、复用或安全处理的部分。

```text
C + D = 1
```

智能不是吞噬数据，而是在真实人类需求、安全约束、成本约束和伦理约束下持续降低 `D`。

## 3. 从失败工程学到神经肥料

失败、错题、工单、AI 跑偏输出、体验反馈和审核记录，是未提纯的工程材料。它们首先属于失败工程学：被收集、脱敏、分类、归因、复盘、生成残差卡。

当这些材料进一步通过授权、审核、验证和可复现评测，它们才可能成为神经肥料。神经肥料不是原始数据，而是被提纯后的残差结构。

## 4. 神经肥料等级 F0-F5

- **F0 Raw Waste / 原始垃圾**：未脱敏、未分类、风险未知，不可训练。
- **F1 Compostable Material / 可堆肥材料**：有潜力但混乱，可进入垃圾站或失败博物馆，不可训练。
- **F2 Structured Fertilizer / 结构化肥料**：已脱敏、有残差卡、完成基础分类，可进入技能候选或缓存候选。
- **F3 Verified Fertilizer / 验证肥料**：已审核，有证据说明能降低某类 `D`，可进入技能胶囊或共享缓存池。
- **F4 Training Candidate Fertilizer / 训练候选肥料**：来源清楚、授权明确、结构完整，有正例、反例、审核记录和验证结果，可进入训练候选池。
- **F5 Foundation-grade Fertilizer / 大模型级肥料**：高质量、多样化、低风险、可复现、可审计，适合公共突触、行业模型或基础模型级候选。

## 5. 九重准入门

1. **Provenance Gate / 来源门**：谁贡献的？什么时候？是否公证？
2. **Consent Gate / 授权门**：能不能训练？能不能派生？能不能公共化？
3. **Privacy Gate / 隐私门**：姓名、手机号、车牌、学校、保单、身份信息是否脱敏？
4. **Residual Clarity Gate / 残差明确门**：它解决哪类 `D`？
5. **Structural Completeness Gate / 结构完整门**：是否包含需求、失败、修正、审核、验证？
6. **Quality Gate / 质量门**：是否重复、低质、幻觉、伪造、刷数据？
7. **Safety Gate / 安全门**：是否涉及违法、侵权、作弊、伤害优化、高风险自动化？
8. **Verification Gate / 验证门**：是否证明它真的降低 `D`？
9. **Reproducibility Gate / 可复现门**：是否有 benchmark、测试集、反例和失败条件？

## 6. 星云突触网络

星云突触网络不是让全网 CPU 一起训练一个完整大模型。它是一个分布式地生产、训练、验证、共享“小突触”的网络。

We do not train one giant brain everywhere. We grow verified synapses everywhere.

## 7. 什么是突触

突触可以是技能突触、残差识别突触、路由突触、评测突触、LoRA 突触、Embedding 突触、记忆突触、安全突触和人类安置突触。

突触不是未经审核的模型补丁。每个突触必须有来源、授权、隐私状态、训练数据摘要、评测结果、适用场景、失效条件、贡献链、版本和回滚点。

## 8. 分布式节点类型

- **Personal Node / 个人节点**：脱敏、分类、残差卡生成、私有缓存。
- **CPU Node / CPU 节点**：去重、打标签、轻量评测、轻量分类器、小型 adapter 低优先级训练、哈希、公证、账本。
- **GPU Node / GPU 节点**：LoRA、Adapter、蒸馏、大批量 embedding、模型评测。
- **Expert Node / 专家节点**：审核、标注、风险判断、失败归因。
- **Commons Node / 公共节点**：合并、版本管理、公共评测、压缩证明、候选发布。

## 9. 突触铸造厂

突触铸造厂接收 F4/F5 肥料候选，并在 mock 流程中把候选分配给 CPU、GPU、专家或公共节点。铸造厂只产生候选突触胶囊，不绕过评测、不绕过人类审核、不执行真实训练。

## 10. 柔性积木模型

未来模型不一定是单体巨兽，而可以是柔性的乐高式模型系统：

```text
Base Core
+ Skill Blocks
+ Residual Blocks
+ Router Blocks
+ Evaluation Blocks
+ Memory Blocks
+ Safety Blocks
+ Domain Blocks
```

大模型用参数记住世界。认知水母用结构组织世界。积木模型网让参数、结构、缓存、技能和评测共同组成柔性智能。

## 11. 星链突触计划

星链突触计划的目标不是发射物理卫星，而是建立一个分布式认知星座：每一个用户设备是微小感知点，每一个团队缓存是局部神经团，每一个行业插件是领域器官，每一个 CPU 节点是分解者，每一个 GPU 节点是突触铸造炉，每一个专家节点是价值感受器，每一个公共技能池是共享星云，每一个开源协议是星点之间的轨道规则。

## 12. 星链突触计划的四层架构

1. **边缘残差层**：个人设备、本地水母、私有缓存、游戏体验、教育错题、工业工单、客户异议。
2. **神经肥料层**：脱敏、结构化、审核、验证后的 F2-F5 肥料。
3. **突触铸造层**：CPU/GPU/专家/公共节点生成小突触。
4. **柔性模型层**：模型积木、技能积木、路由积木、评测积木、安全积木组合成柔性智能。

## 13. 微光剪枝与突触治理

突触会生长，也必须剪枝。否则系统会无限膨胀，变成垃圾山。

微光剪枝动作包括 Keep、Merge、Downgrade、Dormant、Compost、Quarantine、Forget。它不是删除一切，而是在保留历史价值、尊重撤回授权、隔离风险、合并重复、堆肥低质材料之间做治理。

## 14. 与认知水母现有系统的关系

NFAP 连接 ResidualDigestor、OsmoticPrivacyMembrane、CnidocyteSafetyGate、SwarmCommons、PolypMedusaLifecycle、Polyphonic Versioning、World Experience Gateway 和 Human Transition Layer。

残差卡是输入结构，训练候选池是受控出口，突触胶囊是候选产物，模型积木是可评测组件，微光剪枝是生命周期治理。

## 15. 为什么这可能降低大模型参数依赖

如果大量问题可以由结构化残差、缓存、路由、评测器、小适配器和领域积木解决，系统不必把所有知识都压入单一大模型参数中。参数负责泛化，结构负责治理，缓存负责复用，评测负责约束，突触负责局部能力。

这不意味着大模型不重要，而是让大模型不再独自承担全部世界记忆。

## 16. 伦理边界

- 原始数据默认私有。
- 没有授权，不进入训练。
- 未脱敏，不进入训练。
- 未验证，不进入大模型训练候选。
- 用户可以撤回授权。
- 未成年人数据默认 high privacy。
- 企业敏感数据默认 private。
- 训练候选必须有贡献链。
- 分布式训练节点不能私自保留原始数据。
- 公共突触必须可审计、可回滚、可隔离。
- 训练不是吞噬贡献者，而是记录贡献者。
- 训练候选不等于承诺收益。

## 17. 开源路线

v0.4 只落地协议、类型、mock 流程、文档、页面和测试。后续开源路线应先完善评测、审计、撤回、剪枝和安全隔离，再讨论真实 LoRA、Adapter、Embedding 或蒸馏训练。

## 18. MVP 路线图

1. 完成 NFAP 文档、类型、测试和 UI。
2. 将残差卡转换为 `NeuralFertilizer`。
3. 建立训练候选池和 mock 节点分配。
4. 生成突触胶囊和模型积木候选。
5. 增加微光剪枝治理。
6. 为真实训练系统预留人工审核、评测、回滚和撤回接口。

## 19. 结语

认知水母最初只是一个垃圾站：回收失败、错题、工单、AI 跑偏输出。
后来它成为残差操作系统：把失败转成残差卡，把经验变成技能，把技能放入缓存池。
再后来，它成为认知城市：让愿望、贡献、审核、幻化、安置、游戏世界和人类时间都进入循环。
现在，星链突触计划进一步提出：

人类失败提纯出的高质量肥料，可以生长为分布式突触；这些突触可以连接成星云网络；星云网络可以反哺模型、工具、世界体验和认知公地。

最终，我们不追求一个中心神脑吞掉所有东西。
我们追求一张柔性的、可审计的、可剪枝的、可继承的星云突触网络。
