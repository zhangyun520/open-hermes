# Nebula Synapse Network / 星云突触网络

## 1. Concept

The Nebula Synapse Network is not a global CPU swarm training one complete foundation model. It is a network for distributed production, training, verification, and sharing of small synapses.

We do not train one giant brain everywhere.
We grow verified synapses everywhere.

我们不是让全网训练一个巨型脑。
我们是在全网生长被验证的小突触。

## 2. What is a Synapse?

1. Skill Synapse / 技能突触
2. Residual Synapse / 残差识别突触
3. Routing Synapse / 路由突触
4. Evaluation Synapse / 评测突触
5. LoRA Synapse / 低秩适配器突触
6. Embedding Synapse / 向量突触
7. Memory Synapse / 缓存突触
8. Safety Synapse / 安全突触
9. Human Transition Synapse / 人类安置突触

## 3. Node Types

### Personal Node / 个人节点

- 脱敏
- 分类
- 残差卡生成
- 私有缓存

### CPU Node / CPU 节点

- 去重
- 打标签
- 轻量评测
- 轻量分类器
- 小型 adapter 低优先级训练
- 哈希、公证、账本

### GPU Node / GPU 节点

- LoRA
- Adapter
- 蒸馏
- 大批量 embedding
- 模型评测

### Expert Node / 专家节点

- 审核
- 标注
- 风险判断
- 失败归因

### Commons Node / 公共节点

- 合并
- 版本管理
- 公共评测
- 压缩证明
- 候选发布

## 4. Flow

用户投喂失败/愿望/体验反馈
→ 隐私门
→ 脱敏
→ 残差卡
→ 审核
→ 肥料分级
→ 训练候选池
→ 分配给 CPU/GPU/专家节点
→ 训练小突触
→ 评测验证
→ 偏见收容所检查
→ 公证记录版本
→ 进入共享突触池
→ 未来任务命中
→ 贡献回流

## 5. Safety

所有突触必须有：

- 来源
- 授权
- 隐私状态
- 训练数据摘要
- 评测结果
- 适用场景
- 失效条件
- 贡献链
- 版本
- 回滚点
