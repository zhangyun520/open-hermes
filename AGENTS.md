# AGENTS.md

## 项目核心原则

Cognitive Jelly 是一个认知城市操作系统原型。
本项目不采用单线性版本观，而采用复调式并行版本观。

任何重要功能修改，都应考虑：

1. Stable 线：
最小可用、低风险、测试通过。

2. Experimental 线：
大胆探索，但默认不进入公共层。

3. Safety 线：
隐私、权限、人类安置、外部动作风险。

4. Aesthetic 线：
水母 UI、认知大陆、幻化、DIY、体验表达。

5. Low-cost 线：
缓存命中、低模型成本、规则替代模型。

6. Community 线：
贡献、公证、共享缓存和治理。

7. World-model 线：
不确定性、验证、反馈和风险边界。

8. Human-transition 线：
自动化影响人类岗位、身份、收入、尊严时必须审查。

## Codex 输出要求

每次完成任务后必须说明：

- 本次修改属于哪个声部
- 是否影响 safety
- 是否影响 public cache
- 是否影响人类安置
- 是否影响外部连接
- 是否需要人工审核
- 是否可生成 VersionNode
- 如何测试

## 禁止事项

- 不要实现可交易 token
- 不要保存真实 API key
- 不要把隐私数据进入 public cache
- 不要自动执行高风险外部动作
- 不要让 AI 生成版本绕过人工审核直接成为 canonical
- 不要让外观、幻化、付费装饰影响贡献权重
- 不要复刻未经授权版权角色或真人形象
- 不要把世界体验接口做成外挂、代练、刷分或反作弊绕过工具
- 不要把虚拟世界反馈直接当作真实世界结论
