# GOVERNANCE.md

## Governance Goals

- Protocols stay open.
- Data remains layered by privacy and consent.
- Public Skill Commons stays auditable.
- Contribution graphs stay traceable.
- Bias Shelter alerts can be triggered and reviewed.
- High-risk objects can be quarantined.
- AI-generated versions cannot become canonical without human review.

## Decision Areas

- Protocol changes
- Public skill promotion
- Cache object quarantine
- Bias shelter alerts
- Human transition disputes
- Licensed persona / IP governance
- World experience connector approval
- Polyphonic version canonical release

## Roles

Jellyfish governance follows **wide entrance, narrow trunk**: many people can propose and test ideas, but the reviewed project trunk stays small, accountable, and privacy-preserving.

- **Visitor** — reads, tries demos, opens questions, and reports confusing areas.
- **Contributor** — submits ideas, failure cases, benchmark scenarios, code, documentation, research notes, or product design.
- **Triager** — labels issues, checks templates, requests redaction, routes high-risk items to human review, and links duplicates.
- **Maintainer** — reviews PRs, merges low-risk changes, keeps tests passing, and updates contribution records.
- **Core Maintainer** — owns release readiness, protocol compatibility, security-sensitive decisions, and canonical branch protection.
- **Steward Council** — handles governance disputes, public commons promotion, safety escalations, and long-term project direction.

Role changes should be based on sustained contribution, safety judgment, and public accountability rather than status, payment, or follower count.

## Decision Principles

1. Safety gates can block release even when a feature is useful.
2. Public cache promotion requires provenance, consent, anonymization, and review.
3. Human-transition disputes should be reviewed before automation scales.
4. World-experience connectors require authorization and must not bypass anti-cheat.
5. Canonical releases are not the only valid versions; polyphonic branches can coexist.

## Conflict Resolution

When contributors disagree, prefer documented tradeoffs, versioned proposals, and reversible decisions. Use safety review for privacy, abuse, human transition, and external-action risk.
