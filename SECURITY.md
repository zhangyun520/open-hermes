# SECURITY.md

## Sensitive Data

The following are high privacy by default:

- Student names, schools, grades
- Customer phone numbers, license plates, insurance policies
- Government IDs, medical records, financial information
- Enterprise ticket originals, industrial parameters
- API keys
- Unauthorized chat logs
- Minor data

## Reporting Security Issues

Please report security and privacy issues to:

security@example.com

If this repository later publishes a real security contact, use that contact instead.

## Security Principles

- Do not commit secrets.
- Do not expose API keys to frontend code.
- Do not place private data in public cache.
- High-risk actions require human approval.
- External integrations require consent.
- Game connectors must not bypass anti-cheat.
- Persona connectors require authorization.
- Minors' data is private/high sensitivity by default.
- AI-generated versions cannot become canonical without human review.
